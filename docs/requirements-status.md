# WorkLogLite 需求与状态
最后更新：`2026-06-16`

本文档只记录当前仓库对应的真实状态，不保留已经下线或回退的历史结论。

## 产品定位

`WorkLogLite` 当前定位为：
- 原生 `Windows` 离线工作日志工具
- 单机使用
- 可便携分发
- 兼顾工作记录和轻量办公工具区

## 当前稳定能力

### 1. 日志主流程
已实现：
- 按日历选择日期
- 查看当前日期可见条目
- 新增、编辑、保存、删除条目
- 批量删除选中条目
- 列表支持按开始日期、分类、状态排序
- 保存、重载、删除后恢复为单选，不再残留异常多选

### 2. 富文本编辑
已实现：
- `RichEdit` 正文编辑
- 基础格式工具栏
- 编辑区最大化
- `Ctrl + 滚轮` 缩放正文显示
- `Ctrl + 0` 恢复默认缩放
- 时间分隔行插入
- 手动保存和自动保存后保留当前光标与选区

当前约束：
- 仍然是富文本编辑器，不是实时 Markdown 所见即所得
- 仅保留低风险的一键 Markdown 草稿/样式辅助，不做实时语法驱动

### 3. 材料路径
已实现：
- 所有类型条目都可以设置材料路径
- 设置/清除材料路径后立即持久化
- 打开材料目录失败时明确提示
- 条目优先使用自身材料路径
- 无条目级路径时再回退到默认材料目录逻辑

### 4. 工作区
已实现：
- 截图
- 常用回复
- 复制时间戳
- 专注计时
- 单位/耗时计算器
- 数据目录 / 截图目录 / 录音目录 / 日志目录
- 材料根目录
- 便签
- 取色器
- 纯贴
- 时间分隔
- 录系统声
- 录麦克风
- 模板

### 5. 模板
已实现：
- 模板定义文件：`data/templates.txt`
- 模板入口在工作区
- 使用模板时不再操作主编辑区
- 选择模板后会在 `data/template_drafts/` 下生成临时草稿文件
- 草稿会自动替换日期变量并直接打开，适合周报、年报、会议纪要等场景

### 6. 录音
已实现：
- 录系统声与录麦克风
- 保留保存对话框
- 默认文件名按时间生成
- 最长 `5` 小时
- 超时自动停止
- 取消保存时直接丢弃临时录音文件，不保留默认录音文件
- 输出 `WAV`

### 7. 计算器
已实现：
- 单位换算
- 单位前缀统一按 `1024` 处理
- 常用场景改成“数据量 + 带宽 -> 计算耗时”
- 结果输出秒、分钟、小时、天和 `HH:MM:SS`
- 默认窗口尺寸和最小尺寸已调大，首开应可完整显示

### 8. 数据管理
已实现：
- 导出全部数据
- 导入全部数据
- 清空全部数据
- 自动备份旧数据
- 危险路径拦截
- 重载失败时明确提示，不再静默保留旧内存状态

### 9. 报表
已实现：
- Markdown 报表
- 按分类报表
- CSV 导出

## 已明确下线项

- 密码登录 / 改密功能已取消
- “未开始”状态已从 UI 下线

## 近期稳定性修复

最近已经补上的关键问题：
- 自动保存不再把编辑区错误切回空白
- 自动保存后正文缩放比例保持不变
- 自动保存和手动保存后保留当前光标位置
- 删除、分类保存、自动保存的一致性和回滚保护加强
- 导入/清空后的重载失败提示加强
- 列表排序后自动保存会正确重排
- 保存后不再残留异常多选状态
- 材料路径对所有条目即时持久化
- 录音取消保存时不再保留临时文件
- 自动保存失败现在有明确提示
- 取色器文案和错误提示已恢复正常中文

## 剩余风险

当前剩余风险主要偏人工回归，不是编译级风险：
- `screenshot_tool.cpp` 仍有一批历史编码字符串，建议后续单独清理
- 截图、录音、取色这些依赖系统能力的功能，仍需要在目标机器人工回归
- “跨天聚合条目”的删除/回写链路仍值得继续补更细的自动化测试

## 发布建议

当前版本可以继续作为正式发布版本使用。

发布前人工最少回归建议：
1. 新建、保存、排序、删除、批量删除一轮
2. 设置/清除材料路径一轮，覆盖普通条目和任务条目
3. 验证录音、截图、取色各一次
4. 验证导入、导出、清空数据各一次
5. 验证模板草稿创建、计算器耗时计算各一次
6. 验证便携包解压后直接运行
