# WorkLogLite 需求与实现状态

最后更新: 2026-04-24

本文用于沉淀当前已经明确的产品需求、当前实现状态、验证情况和后续待确认项，避免继续依赖聊天记录追溯。

- 已实现: 代码已接入，且当前构建/测试通过
- 已实现待人工回归: 代码已接入，但更依赖 GUI 手工验证

## 当前共识需求

### 1. 日期语义统一

需求:

- 各类带开始/结束的对象按“日期”处理，不按具体时分秒处理
- 开始日期默认当前日期
- 结束日期必须大于开始日期

当前状态: 已实现

已落地内容:

- 主界面编辑区开始/结束改为日期控件
- 长期任务编辑区开始/结束改为日期控件
- 周期会议编辑区开始/结束改为日期控件
- 保存前统一校验“结束日期必须晚于开始日期”
- 数据层也增加了非法日期范围拦截，避免坏数据落盘

说明:

- 旧数据中的 `HH:MM` 时间格式仍兼容读取
- 主路径写入与展示已经统一为日期语义

### 2. 每个长期任务绑定自己的材料目录

需求:

- 每个长期任务都可以指定自己的材料目录
- 创建任务时就可以指定目录
- 编辑任务时也可以修改目录
- 查看任务时支持直接打开该目录

当前状态: 已实现待人工回归

已落地内容:

- 长期任务模型包含独立 `materials_dir`
- 长期任务编辑区包含材料目录显示、设置路径、打开目录
- 新建任务未保存前也会先生成草稿任务 ID，支持提前设置/打开目录
- 任务材料目录支持自动回落到默认任务材料根目录

限制与约束:

- 禁止使用 UNC 网络共享路径
- 若目录不在 `data` 目录内，会给出导入导出不可携带的提示

### 3. 长期任务列表显示材料列

需求:

- 管理长期任务窗口左侧列表显示 `任务` 和 `材料` 两列
- 材料列展示路径
- 点击材料列可快速打开目录

当前状态: 已实现待人工回归

已落地内容:

- 长期任务左侧列表已改为 `ListView`
- 已增加 `任务 / 材料` 两列
- 材料路径会直接显示在列表中
- 双击材料列会打开对应目录

说明:

- 右键菜单也保留了打开材料目录/材料根目录的能力

### 4. 主界面列表显示目录列

需求:

- 主界面右侧日志列表增加目录列
- 任务进展优先显示任务材料目录
- 普通条目显示条目自己的材料目录，未设置时回落到默认工作材料目录

当前状态: 已实现待人工回归

已落地内容:

- 主界面列表当前列为 `日期范围 / 分类 / 标题 / 状态 / 目录`
- 第 5 列显示目录路径
- 双击目录列可直接打开目录

### 5. 办公区按钮布局调整

需求:

- 左下办公区保留常用按钮
- 只移除材料目录相关按钮
- 其他已有按钮保留

当前状态: 已实现

当前办公区保留项:

- 截图
- 常用回复
- 复制时间戳
- 专注
- 计算器
- 数据目录
- 截图目录
- 取色器
- 纯贴
- 录系统声
- 录麦克风

已移除项:

- 任务材料
- 材料目录
- 设置路径

### 6. 取色器

需求:

- 取色范围必须是全屏
- 鼠标移动时实时显示颜色
- 左键确认复制颜色值
- 结果至少支持 `#RRGGBB`

当前状态: 已实现待人工回归

已落地内容:

- 已增加独立取色器模块
- 办公区已接入 `取色器` 按钮
- 取色器按全屏覆盖方式工作
- 选中后复制十六进制颜色到剪贴板

说明:

- 该功能更依赖不同显示器/DPI/缩放环境下的手工验证

### 7. 纯文本粘贴

需求:

- 提供一个将剪贴板文本按纯文本插入正文的快捷入口

当前状态: 已实现

已落地内容:

- 办公区已接入 `纯贴` 按钮
- 会读取剪贴板文本并以纯文本插入当前正文 RichEdit

### 8. 录音功能

需求:

- 支持分别录制系统声音和麦克风声音
- 点击录制后显示时长
- 停止时弹出保存窗口，支持命名
- 录音文件要能被系统原生播放器打开播放
- 默认命名清晰、可检索
- 录音按钮中文化，录制中有明显红色录制状态

当前状态: 已实现待人工回归

已落地内容:

- 办公区已增加 `录系统声` / `录麦克风`
- 录制中按钮显示时长
- 停止时弹出保存窗口
- 保存后会调用系统默认播放器打开
- 输出格式统一为兼容性更强的 `16-bit PCM WAV`
- 修复了 WAV 头、时长、静音、丢包、加速等问题
- 默认命名规则已优化，包含来源与时间戳
- 按钮已中文化，录制中有红色录制态

说明:

- 此项自动化测试无法覆盖真实声卡/默认播放设备
- 需要在目标机器上分别手工回归系统声和麦克风录制

### 9. 导入 / 导出 / 清空数据

需求:

- 支持完整导出全部数据
- 支持从外部目录导入并替换当前数据
- 支持清空数据并保留备份
- 失败时要可回退，不应破坏当前数据

当前状态: 已实现

已落地内容:

- 已抽离 `data_ops` 模块处理整目录导出/导入/清空
- 导入支持直接选择 `data` 根目录或外层包装目录
- 导入会先复制到临时目录，再执行安全切换
- 清空数据前会创建备份目录
- Windows 下目录移动失败时，会走更稳的备份/清空兜底逻辑
- 明确拦截把当前数据目录当作导入源、把导出目录放到当前数据目录内部等危险操作

### 10. 数据层健壮性与校验

需求:

- 避免坏数据落盘
- 对任务、条目等关键模型进行基础校验

当前状态: 已实现

已落地内容:

- 日记录保存时校验开始/结束日期范围
- 长期任务保存时校验 `id`、标题、日期范围
- 报表与读取逻辑兼容旧数据

### 11. 窗口 UI 首屏可见

需求:

- 打开窗口后应默认完整可见
- 不应依赖用户手工拖拽放大才能正常使用

当前状态: 部分实现

已落地内容:

- 长期任务窗口已做更大的默认宽度与布局适配
- 多处管理窗口增加了最小尺寸/工作区内显示约束

仍需关注:

- 这类问题高度依赖屏幕分辨率、DPI、系统字体缩放
- `管理长期任务`、`管理分类`、`管理周期会议` 等窗口仍建议继续人工回归

### 12. 中文文案与乱码

需求:

- 运行时 UI 不应出现乱码
- 常用按钮与录音入口应中文化

当前状态: 部分实现

已落地内容:

- 新接入和近几轮修复的关键交互文案已尽量恢复正常中文
- 录音按钮、导入导出/清空关键提示、部分日期与材料目录提示已恢复

已知情况:

- 仓库中部分旧源码字符串历史上存在编码污染痕迹
- 终端查看源码时也可能因为代码页问题出现乱码
- 仍建议继续针对各窗口逐页清理遗留文案

## 建议的人工回归清单

建议每次发布前至少手工过一遍以下项目:

1. 主界面新建条目，确认开始/结束为日期控件，且结束日期不能早于或等于开始日期
2. 主界面列表确认存在 `目录` 列，双击该列可打开目录
3. 办公区确认只移除了材料目录相关按钮，其余常用按钮仍在
4. 取色器确认在当前屏幕/DPI 下能正确取色并复制 `#RRGGBB`
5. `纯贴` 确认插入正文时不带格式
6. 长期任务窗口确认左侧存在 `任务 / 材料` 两列
7. 长期任务新建时确认右侧可直接设置材料目录
8. 双击长期任务材料列确认能打开目录
9. 分别验证 `录系统声` 和 `录麦克风`，确认保存后的 WAV 可被系统播放器打开
10. 验证导出、导入、清空数据三条链路的弹窗、备份目录和结果提示
