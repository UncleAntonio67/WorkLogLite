# WorkLogLite 需求与状态

最后更新：`2026-06-12`

本文档只记录当前仓库对应的真实状态，不保留已经下线或回退的历史结论。

## 产品定位

`WorkLogLite` 当前定位为：

- 原生 `Windows` 离线工作日志工具
- 单机使用
- 可便携分发
- 兼顾工作记录和轻量办公工具区

## 当前稳定能力

### 1. 日志主流程
已实现：
- 按日历选择日期
- 查看当前日期可见条目
- 新增、编辑、保存、删除条目
- 批量删除选中条目
- 条目列表按日期范围、分类、状态排序
- 保存 / 重载 / 删除后恢复单选，不再残留异常多选

### 2. 富文本编辑
已实现：
- `RichEdit` 正文编辑
- 基础格式工具栏
- 编辑区最大化
- `Ctrl + 滚轮` 调整正文显示缩放
- `Ctrl + 0` 恢复默认缩放
- 时间分隔行插入

当前约束：
- 仍然是富文本编辑器，不是实时 Markdown 所见即所得
- Markdown 仍保留低风险的一键转换能力

### 3. 材料路径
已实现：
- 所有类型条目都可以设置材料路径
- 设置 / 清除材料路径后立即持久化
- 条目优先使用自身材料路径
- 无条目级路径时，再回退到默认材料目录逻辑
- 打开材料目录失败时明确提示

### 4. 工作区
已实现：
- 截图
- 常用回复
- 复制时间戳
- 专注计时
- 单位 / 带宽计算器
- 数据目录 / 截图目录 / 录音目录 / 日志目录
- 材料根目录
- 便签
- 取色器
- 纯贴
- 时间分隔
- 录系统声
- 录麦克风

### 5. 录音
已实现：
- 录系统声与录麦克风
- 保留保存对话框
- 默认文件名按时间生成
- 最长 `5` 小时
- 超时自动停止
- 输出 `WAV`

### 6. 数据管理
已实现：
- 导出全部数据
- 导入全部数据
- 清空全部数据
- 自动备份旧数据
- 危险路径拦截
- 重载失败时明确提示，不再静默保留旧内存状态

### 7. 报表
已实现：
- Markdown 报表
- 按分类报表
- CSV 导出

## 已明确下线项

- 密码登录 / 改密功能已取消
- “未开始”状态已从 UI 下线

## 自动化测试现状

当前 `smoke tests` 已覆盖：
- 分类文件读写
- 日记录 `roundtrip`
- `status / materials_dir / body_rtf_b64` 持久化
- 非法日期范围拒绝
- 旧 `.wlmd` 兼容读取
- 任务数据 `roundtrip`
- 报表生成链路
- demo 数据生成
- 导入、导出、清空数据
- 坏 `.wlr` / `.wlt` 文件容错
- 非法导入目录拒绝
- 未知任务状态回落到 `none`

当前结论：
- `build.bat --no-pause` 通过
- `test.bat --no-pause` 通过

## 近期稳定性修复

最近已经补上的关键问题：
- 自动保存不再把编辑区错误切回空白
- 自动保存后正文缩放比例保持不变
- 删除、分类保存、自动保存的回滚和一致性加强
- 导入 / 清空后的重载失败提示
- 列表排序后自动保存会正确重排
- 保存后不再残留异常多选状态
- 材料路径对所有条目即时持久化
- 系统打开入口失败提示加强

## 剩余风险

当前剩余风险主要偏人工回归，不是编译级风险：
- 截图工具源码仍有历史编码污染，需要单独清理
- 录音、截图、取色这些依赖系统能力的功能，仍需在目标机器人工回归
- “跨天聚合条目”的删除 / 回写链路还值得继续补更细的自动化测试

## 发布建议

当前版本可继续作为正式发布版本使用。

发布前人工最少回归建议：
1. 新建、保存、排序、删除、批量删除一轮
2. 设置 / 清除材料路径一轮，覆盖普通条目和任务条目
3. 验证录音、截图、取色各一次
4. 验证导入、导出、清空数据各一次
5. 验证便携包解压后直接运行
